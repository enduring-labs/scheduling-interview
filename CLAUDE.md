# Project Guide

## Stack

- **Next.js 15** (App Router, React Server Components, server actions)
- **Drizzle ORM** + **PostgreSQL**
- **Tailwind CSS v4**

## Commands

```bash
pnpm dev          # start dev server
pnpm db:generate  # generate migration after schema changes (ALWAYS run this)
pnpm db:migrate   # apply pending migrations to the database
pnpm db:seed      # insert seed data
pnpm db:reset     # drop all tables, then run: pnpm db:migrate && pnpm db:seed
```

---

## IMPORTANT: How to Change the Database Schema

> **Never edit migration SQL files by hand. Never write migration SQL yourself.**
> Drizzle generates migrations automatically from the schema file.

### The only correct workflow is:

**1. Edit the schema file**

All table definitions live in one file:

```
lib/db/schema.ts
```

Make your changes here — add a table, add a column, change a type, etc.

**2. Run generate**

```bash
pnpm db:generate
```

This reads `lib/db/schema.ts`, diffs it against the last migration, and writes a new SQL file + updates the migration journal automatically. You do not touch the output.

**3. Apply the migration**

```bash
pnpm db:migrate
```

**4. If starting fresh (e.g. schema is broken or you want clean state)**

```bash
pnpm db:reset
pnpm db:migrate
pnpm db:seed
```

### What NOT to do

- Do **not** create or edit files in `drizzle/` manually
- Do **not** edit `drizzle/meta/_journal.json` manually
- Do **not** skip `pnpm db:generate` and go straight to `pnpm db:migrate` — the migration file won't exist yet

## Project Structure

```
app/
  page.tsx              # home / dashboard
  dispatch/page.tsx     # dispatcher view — assign techs to work orders
  tech/page.tsx         # technician view — see assigned work orders
  tenant/
    page.tsx            # tenant view — see own work orders (?phone=+1...)
    new/page.tsx        # tenant form — submit a new work order

lib/db/
  schema.ts             # ALL table definitions (edit this to change the schema)
  index.ts              # db client (do not edit)
  seed.ts               # seed data
  reset.ts              # drop + reset script

drizzle/                # auto-generated by drizzle-kit (do not edit manually)
  *.sql                 # migration files
  meta/                 # migration journal and snapshots
```

## Database Tables

| Table | Purpose |
|---|---|
| `tenants` | People who rent units and submit work orders |
| `technicians` | Field techs who are assigned to and complete work orders |
| `work_orders` | Maintenance requests — links a tenant, a property, and optionally a technician |

## URL Conventions

- Tenant portal: `/tenant?phone=+15552001001`
- Tech portal: `/tech?id=<uuid>`
- Dispatch: `/dispatch`
